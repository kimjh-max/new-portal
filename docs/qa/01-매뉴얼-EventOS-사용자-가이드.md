## 1. 시스템 개요

### 1.1 EventOS란?

EventOS는 EpicStage의 **행사/공연 기획 통합 운영 포털**입니다. 프로젝트 생성부터 견적, 일정 관리, 지출결의, 인건비, 구매요청, 보고서 작성까지 행사 기획의 전 과정을 하나의 웹 애플리케이션에서 처리할 수 있습니다.

### 1.2 기술 사양

| 항목 | 내용 |
|------|------|
| 애플리케이션 유형 | 단일 페이지 애플리케이션 (SPA) |
| 프레임워크 | React 18 |
| 호스팅 | GitHub Pages |
| 권장 브라우저 | Google Chrome (최신 버전) |
| 호환 브라우저 | Edge, Safari, Firefox (최신 버전) |

### 1.3 데이터 저장 구조

EventOS는 데이터 안전성을 위해 **3계층 저장 구조**를 사용합니다.

| 계층 | 저장소 | 저장 시점 | 설명 |
|------|--------|-----------|------|
| 1계층 | localStorage (브라우저) | 즉시 | 모든 변경사항이 즉시 브라우저에 저장됩니다. |
| 2계층 | Cloudflare D1/R2 (클라우드) | 3초 디바운스 | 변경 후 3초 경과 시 클라우드에 자동 동기화됩니다. |
| 3계층 | Google Drive GAS (백업) | 30분 주기 | 30분마다 전체 데이터를 Google Drive에 백업합니다. |

> **참고:** 브라우저의 localStorage를 삭제하면 로컬 데이터가 사라질 수 있으나, 클라우드 동기화 및 Google Drive 백업을 통해 복원할 수 있습니다.

### 1.4 반응형 지원

| 디바이스 | 화면 너비 | 레이아웃 |
|----------|-----------|----------|
| 데스크톱 | 1100px 이상 | 3~4열 그리드, 상단 탭 네비게이션 |
| 태블릿 | 769~1100px | 2열 그리드 |
| 모바일 | 768px 이하 | 1열 그리드, 하단 네비게이션 |

## 2. 로그인 및 인증

### 2.1 로그인 절차

1. 브라우저에서 EventOS URL에 접속합니다.
2. **사번(아이디)** 필드에 사번을 입력합니다.
3. **비밀번호** 필드에 비밀번호를 입력합니다.
4. **로그인** 버튼을 클릭합니다.

> **기본 관리자 계정:** 아이디 `admin` / 비밀번호 `Epicstage2026`

### 2.2 비밀번호 표시/숨김

비밀번호 입력란 우측의 **눈 모양 아이콘**을 클릭하면 입력 중인 비밀번호를 확인하거나 다시 숨길 수 있습니다.

### 2.3 로그인 실패 잠금

보안을 위해 비밀번호를 연속으로 잘못 입력하면 계정이 일시적으로 잠깁니다.

| 항목 | 값 |
|------|-----|
| 최대 실패 횟수 | 5회 |
| 잠금 시간 | 10분 |

5회 연속 실패 시 해당 계정은 **10분간 로그인이 불가**합니다. 10분이 지나면 자동으로 잠금이 해제됩니다.

### 2.4 계정 상태

| 상태 | 설명 |
|------|------|
| 승인(approved) | 정상 사용 가능한 계정 |
| 대기(pending) | 신규 등록 후 관리자 승인 대기 중 |
| 차단(blocked) | 관리자에 의해 사용이 차단된 계정 |

- 신규 계정을 등록하면 **대기(pending)** 상태가 되며, **관리자가 승인**해야 로그인이 가능합니다.
- 차단된 계정은 관리자만 해제할 수 있습니다.

### 2.5 세션 관리

- 로그인 후 **새로고침(F5)을 해도 세션이 유지**됩니다. (sessionStorage 기반)
- **브라우저 탭을 닫으면 자동으로 로그아웃**됩니다.
- 같은 브라우저에서 새 탭을 열면 별도로 로그인해야 합니다.

## 3. 메인 네비게이션 및 공통 기능

### 3.1 메뉴 구성

EventOS는 총 **8개 주요 메뉴**로 구성됩니다.

| 순서 | 메뉴명 | 설명 |
|------|--------|------|
| 1 | 홈 | 대시보드 (KPI, 진행 중 프로젝트, 최근 업무) |
| 2 | 프로젝트 | 프로젝트 목록 및 상세 관리 |
| 3 | 내 업무 | 개인 담당 프로젝트 및 과업 현황 |
| 4 | 지출 | 지출결의서 등록 및 관리 |
| 5 | 인건비 | 인건비 등록 및 원천세 계산 |
| 6 | 구매 | 구매요청 등록 및 처리 |
| 7 | 파일 | 전체 프로젝트 파일 자료실 |
| 8 | 관리 | 관리자 전용 설정 (계정, 기준금액) |

- **데스크톱(1100px 이상):** 화면 상단에 탭바로 표시됩니다.
- **모바일(768px 이하):** 화면 하단에 네비게이션 바로 표시됩니다.

### 3.2 동기화 상태바

화면 상단(또는 하단)에 현재 데이터 동기화 상태가 표시됩니다.

| 상태 | 의미 |
|------|------|
| idle | 대기 중 (변경사항 없음) |
| syncing | 동기화 진행 중 |
| success | 동기화 완료 |
| error | 동기화 오류 발생 |
| offline | 네트워크 연결 끊김 |

> **참고:** `offline` 상태에서도 로컬 작업은 정상적으로 가능하며, 네트워크 복구 시 자동으로 동기화됩니다.

### 3.3 백업 및 복원

#### 수동 백업

1. 상단 네비게이션의 **백업 버튼**을 클릭합니다.
2. 현재 시점의 전체 데이터가 Google Drive에 즉시 백업됩니다.
3. 백업 완료 알림이 표시됩니다.

#### 복원

1. 상단 네비게이션의 **복원 버튼**을 클릭합니다.
2. **최근 3개 백업** 목록이 표시됩니다.
3. 복원할 시점을 선택하고 **복원** 버튼을 클릭합니다.
4. 확인 대화상자에서 **확인**을 클릭하면 해당 시점의 데이터로 복원됩니다.

> **주의:** 복원을 실행하면 현재 데이터가 선택한 백업 시점의 데이터로 **덮어쓰기**됩니다. 복원 전에 현재 데이터를 내보내기로 별도 저장해 두는 것을 권장합니다.

### 3.4 데이터 내보내기/가져오기

#### 데이터 내보내기

1. **데이터 내보내기** 버튼을 클릭합니다.
2. 전체 데이터가 **JSON 파일**로 다운로드됩니다.
3. 다운로드된 파일을 안전한 위치에 보관합니다.

#### 데이터 가져오기

1. **데이터 가져오기** 버튼을 클릭합니다.
2. 파일 선택 대화상자에서 이전에 내보낸 **JSON 파일**을 선택합니다.
3. 가져오기 확인 후 데이터가 복원됩니다.

> **주의:** 가져오기를 실행하면 현재 데이터가 JSON 파일의 데이터로 대체됩니다.

### 3.5 로그아웃

상단 네비게이션의 **로그아웃** 버튼을 클릭하면 즉시 로그아웃되며 로그인 화면으로 돌아갑니다.

## 4. 대시보드

로그인 후 가장 먼저 표시되는 **홈** 화면입니다. 전체 사업 현황을 한눈에 파악할 수 있습니다.

### 4.1 KPI 카드

화면 상단에 4개의 핵심 지표 카드가 표시됩니다.

| KPI 카드 | 설명 |
|----------|------|
| 전체 프로젝트 수 | 시스템에 등록된 전체 프로젝트의 개수 |
| 총 계약금액 | 모든 프로젝트의 계약금액 합계 |
| 평균 수익률 | 전체 프로젝트의 평균 예상 수익률 (%) |
| 완료 프로젝트 | 상태가 '완료'인 프로젝트의 개수 |

### 4.2 진행 중 프로젝트 목록

KPI 카드 아래에 현재 **진행 중인 프로젝트 목록**이 표시됩니다.

- 각 프로젝트 카드에는 프로젝트명, 클라이언트, 상태, 진행률이 표시됩니다.
- **프로젝트 카드를 클릭**하면 해당 프로젝트의 상세 화면으로 이동합니다.

### 4.3 최근 업무

전체 프로젝트에서 **최신 6건의 업무 기록**이 자동으로 표시됩니다. 최근에 변경되거나 추가된 업무를 빠르게 확인할 수 있습니다.

## 5. 프로젝트 관리

### 5.1 프로젝트 목록

**프로젝트** 메뉴를 클릭하면 등록된 전체 프로젝트 목록이 표시됩니다.

#### 검색

- 상단 검색창에 **프로젝트명**, **클라이언트명**, **프로듀서명** 중 하나를 입력하여 검색할 수 있습니다.
- 입력 즉시 필터링되어 결과가 표시됩니다.

#### 상태 필터

검색창 옆의 상태 필터 드롭다운에서 원하는 상태를 선택합니다.

| 필터 옵션 | 설명 |
|-----------|------|
| 전체 | 모든 프로젝트 표시 |
| 진행중 | 현재 진행 중인 프로젝트만 표시 |
| 준비중 | 준비 단계의 프로젝트만 표시 |
| 완료 | 완료된 프로젝트만 표시 |
| 보류 | 보류 중인 프로젝트만 표시 |

### 5.2 프로젝트 카드 (ProjectCard)

각 프로젝트는 카드 형태로 표시되며 다음 정보가 포함됩니다.

- **상태 배지:** 현재 프로젝트 상태를 색상으로 구분하여 표시
- **진행률 바:** 체크리스트 기반 전체 진행률을 시각적으로 표시
- **수익률:** 예상 수익률 (%)
- **행사일:** 행사 예정일
- **태그:** 프로젝트에 설정된 분류 태그

### 5.3 새 프로젝트 생성

1. 프로젝트 목록 화면에서 **새 프로젝트** 버튼을 클릭합니다.
2. 아래 항목을 입력합니다.

| 항목 | 필수 여부 | 설명 |
|------|-----------|------|
| 프로젝트명 | **필수** | 행사/프로젝트의 이름 |
| 클라이언트 | **필수** | 발주처/의뢰사명 |
| 프로듀서 | 선택 | 담당 프로듀서명 |
| 계약금액 | 선택 | 계약 총 금액 (원) |
| 행사일 | 선택 | 행사 예정일 |
| 태그 | 선택 | 분류 태그 |
| 상태 | 선택 | 초기 상태 설정 (기본값: 준비중) |

3. **생성** 버튼을 클릭합니다.
4. 프로젝트가 생성되면 **기준 단가표(priceList)**가 자동으로 해당 프로젝트의 체크리스트에 복사됩니다.

> **참고:** 기준 단가표는 관리자 설정의 **기준 금액 관리**에서 사전에 등록된 항목입니다. 프로젝트 생성 시 이 단가표가 체크리스트의 초기 항목으로 자동 설정되므로, 관리자가 기준 금액을 미리 정리해 두는 것이 좋습니다.

### 5.4 프로젝트 삭제

- **관리자(admin) 권한이 있는 사용자만** 프로젝트를 삭제할 수 있습니다.
- 프로젝트 상세 화면에서 **삭제** 버튼을 클릭합니다.
- **확인 모달**이 표시되며, 확인을 클릭해야 최종 삭제됩니다.

> **주의:** 프로젝트를 삭제하면 해당 프로젝트의 모든 데이터(체크리스트, 간트차트, 업무 로그, 지출결의, 인건비, 구매요청, 파일 등)가 함께 삭제됩니다. 이 작업은 되돌릴 수 없습니다.

## 6. 프로젝트 상세

프로젝트 카드를 클릭하면 프로젝트 상세 화면으로 이동합니다. 상세 화면은 **6개 탭**으로 구성됩니다.

| 탭 | 내용 |
|-----|------|
| 개요 | 과업지시서, AI 요약, 일정/예산 현황 |
| 체크리스트/견적 | 항목별 체크리스트, 견적서 시스템 |
| 간트차트 | 타임라인 기반 일정 관리 |
| 업무 로그 | 코멘트, 과업 위임, 자동 기록 |
| 보고서 | 운영계획안/결과보고서 작성 |
| 파일 | 프로젝트별 파일 업로드/관리 |

---

### 6.1 개요 탭

프로젝트의 전반적인 정보를 확인하고 편집하는 탭입니다.

#### 과업지시서 (TaskOrderPanel)

프로젝트의 핵심 과업 정보를 기록하는 패널입니다.

| 항목 | 설명 |
|------|------|
| 발신인 | 과업을 지시한 사람/부서 |
| 수신일 | 과업지시서 수신 날짜 |
| 장소 | 행사 장소 |
| 참석 규모 | 예상 참석 인원 |
| 목적 | 행사의 목적 |
| 과업 | 수행해야 할 주요 과업 내용 |
| 요구사항 | 클라이언트의 특별 요구사항 |
| 참고 | 기타 참고 사항 |

각 항목을 클릭하여 직접 편집할 수 있으며, 변경사항은 자동으로 저장됩니다.

#### AI 요약

- 체크리스트에 등록된 항목들을 기반으로 **시뮬레이션 요약**을 생성합니다.
- 현재 프로젝트의 전체적인 진행 상황과 주요 사항을 빠르게 파악할 수 있습니다.

#### 일정 현황

| 항목 | 설명 |
|------|------|
| 시작일 | 프로젝트 시작일 |
| 행사일 | 행사 당일 날짜 |
| 종료일 | 프로젝트 종료 예정일 |

#### 예산 현황

| 항목 | 설명 |
|------|------|
| 계약금액 | 클라이언트와의 계약 총 금액 |
| 예상원가 | 체크리스트 기반 예상 비용 합계 |
| 예상수익 | 계약금액 - 예상원가 |

### 6.2 체크리스트/견적 탭

프로젝트의 세부 항목을 관리하고 견적서를 작성하는 핵심 탭입니다.

#### 체크리스트 관리

##### 구분

체크리스트 항목은 4개 구분으로 분류됩니다.

| 구분 | 설명 |
|------|------|
| 기획 | 기획 관련 항목 (콘셉트, 시나리오, 진행 등) |
| 테크니컬 | 기술 장비 항목 (음향, 조명, 영상 등) |
| 제작 | 제작물 항목 (무대, 인쇄물, 소품 등) |
| 기타 | 위 구분에 해당하지 않는 기타 항목 |

##### 항목별 필드

각 체크리스트 항목은 다음 필드를 가집니다.

| 필드 | 설명 |
|------|------|
| 완료 토글 | 체크박스로 완료/미완료 전환 |
| 항목명 | 항목의 이름 |
| 단가 | 항목의 단가 (원) |
| 수량 | 수량 |
| 일자 | 관련 일자 (필요 시) |
| 비고 | 추가 메모 |

- 모든 필드는 **인라인 편집** 방식으로, 해당 필드를 클릭하면 바로 수정할 수 있습니다.
- 변경사항은 즉시 자동 저장됩니다.

##### 항목 추가/삭제

1. **항목 추가:** 해당 구분 섹션 하단의 **+ 항목 추가** 버튼을 클릭합니다.
2. **항목 삭제:** 항목 우측의 **삭제(X)** 버튼을 클릭합니다.

##### 진행률 자동 계산

- 전체 항목 중 완료 체크된 항목의 비율이 자동으로 계산되어 **진행률**로 표시됩니다.
- 이 진행률은 프로젝트 카드 및 대시보드에도 반영됩니다.

#### 견적서 시스템

체크리스트 항목을 기반으로 견적서를 자동 생성합니다.

##### 견적 유형 선택

| 유형 | 적용 대상 | 산출 방식 |
|------|-----------|-----------|
| 일반기업 | 민간 기업 클라이언트 | 단가 x 1.15 (+15% 마진) + 부가세 10% |
| 공공기관 | 공공기관 클라이언트 | A(공급가액) + B(관리비 8%) + C(수익 10%) + D(부가세 10%) |

##### 일반기업 견적 산출

```
공급가액 = 단가 x 수량 x 1.15 (15% 마진 포함)
부가세   = 공급가액 x 0.10
합계     = 공급가액 + 부가세
```

##### 공공기관 견적 산출

```
A. 공급가액 = 단가 x 수량
B. 관리비   = A x 0.08 (8%)
C. 수익     = (A + B) x 0.10 (10%)
D. 부가세   = (A + B + C) x 0.10 (10%)
합계       = A + B + C + D
(10원 단위 절사 적용)
```

> **참고:** 공공기관 견적은 모든 금액에서 **10원 단위 절사**가 적용됩니다.

##### 추가 항목 (customItems)

견적서에만 포함되는 별도 항목을 추가할 수 있습니다.

1. 견적서 화면에서 **추가 항목** 섹션을 찾습니다.
2. **+ 추가** 버튼으로 새 항목을 입력합니다.
3. 항목명, 단가, 수량 등을 기입합니다.
4. 이 항목은 **견적서에만 반영**되며, 기본 체크리스트에는 영향을 주지 않습니다.

##### 기준 예산 절사

목표 금액에 맞추어 견적 총액을 조정하는 기능입니다.

1. **기준 예산 절사** 영역에 **목표 금액**을 입력합니다.
2. 시스템이 자동으로 수익/마진 부분에서 감액하여 **목표 금액에 맞춘 견적**을 생성합니다.
3. 절사된 금액 정보가 표시됩니다.

> **활용 예시:** 클라이언트 예산이 5,000만 원인데 견적 합계가 5,200만 원인 경우, 목표 금액을 5,000만 원으로 입력하면 마진에서 200만 원이 감액되어 5,000만 원에 맞춰집니다.

##### PDF 출력

1. **견적서 PDF** 버튼을 클릭합니다.
2. 구분별 테이블, 합계, 절사 정보가 포함된 견적서가 PDF로 생성됩니다.
3. 브라우저의 인쇄/다운로드 대화상자에서 저장합니다.

##### 견적 이력

- 견적서를 생성할 때마다 **스냅샷**이 자동으로 저장됩니다.
- **견적 이력** 목록에서 과거에 생성한 견적을 확인할 수 있습니다.
- 과거 견적의 **PDF 재출력**이 가능합니다.

### 6.3 간트차트 탭

프로젝트 일정을 타임라인 형태로 시각적으로 관리합니다.

#### 타임라인 보기

| 요소 | 설명 |
|------|------|
| 월별 헤더 | 상단에 월 단위로 구분된 타임라인 표시 |
| D-day 표시 | 행사일까지 남은 일수 표시 |
| 오늘 표시 | 현재 날짜 위치에 세로선 표시 |
| 주말 구분 | 토요일/일요일을 배경색으로 구분 |

#### 과업 추가

1. **+ 과업 추가** 버튼을 클릭합니다.
2. 아래 항목을 입력합니다.

| 항목 | 설명 |
|------|------|
| 구분 | 과업 분류 (기획/테크니컬/제작/기타) |
| 과업명 | 과업의 이름 |
| 시작일 | 과업 시작 예정일 |
| 종료일 | 과업 종료 예정일 |
| 색상 | 타임라인 바의 색상 지정 |
| 진행률 | 현재 진행률 (0~100%) |

3. **저장**을 클릭하면 타임라인에 과업 바가 표시됩니다.

#### 세부항목 (Subtask)

각 과업 하위에 세부 작업을 추가할 수 있습니다.

1. 과업 항목을 클릭하여 상세를 펼칩니다.
2. **+ 세부항목 추가** 버튼을 클릭합니다.
3. 세부항목명을 입력합니다.
4. 세부항목의 **완료 토글**로 완료 여부를 관리합니다.

#### 체크리스트 가져오기

체크리스트 탭에 등록된 항목들을 간트차트 과업으로 자동 변환합니다.

1. **체크리스트 가져오기** 버튼을 클릭합니다.
2. 체크리스트의 구분(기획/테크니컬/제작/기타)별로 과업이 자동 생성됩니다.
3. 생성된 과업의 시작일/종료일을 필요에 따라 조정합니다.

#### 날짜 메모

- **타임라인의 특정 날짜**에 메모를 추가할 수 있습니다.
- **개별 항목**에도 날짜별 메모를 기록할 수 있습니다.
- 날짜 메모는 해당 날짜 위에 마우스를 올리면 확인할 수 있습니다.

### 6.4 업무 로그 탭

프로젝트의 업무 진행 기록을 관리하는 탭입니다.

#### 코멘트 입력

1. 텍스트 입력란에 업무 내용을 작성합니다.
2. **등록** 버튼을 클릭하거나 **Enter 키**를 누릅니다.
3. 현재 시간과 작성자 정보와 함께 로그가 기록됩니다.

> **빠른 입력:** Enter 키를 누르면 즉시 코멘트가 등록되므로, 여러 건의 로그를 빠르게 입력할 수 있습니다.

#### 과업 위임

다른 팀원에게 과업을 위임하고 기록합니다.

1. **과업 위임** 영역에서 **대상**을 선택합니다 (팀원 목록에서 선택).
2. **과업명**을 입력합니다.
3. **전달사항**을 입력합니다 (선택).
4. **위임** 버튼을 클릭합니다.
5. 위임 내역이 업무 로그에 자동으로 기록됩니다.

> **참고:** 위임된 과업은 대상자의 **내 업무** 화면에도 표시됩니다.

#### 자동 기록

다음과 같은 변경이 발생하면 업무 로그에 **자동으로 기록**됩니다.

- 체크리스트 항목 완료/미완료 전환
- 체크리스트 항목 추가/삭제
- 기타 주요 데이터 변경

### 6.5 보고서 탭 (DocEditorPanel)

행사 운영계획안 및 결과보고서를 작성하고 출력하는 탭입니다.

#### 서브탭

| 서브탭 | 설명 |
|--------|------|
| 운영계획안 (plan) | 행사 사전 운영 계획서 작성 |
| 결과보고서 (report) | 행사 종료 후 결과 보고서 작성 |

#### 6개 섹션

각 보고서(운영계획안/결과보고서)는 6개 섹션으로 구성됩니다.

| 섹션 | 영문명 | 주요 내용 |
|------|--------|-----------|
| 개요 | Overview | 행사 개요, 목적, 일시, 장소 등 기본 정보 |
| 장소 | Venue | 행사장 정보, 배치도, 접근성 등 |
| 디자인 | Design | 무대 디자인, 시각 요소, 브랜딩 등 |
| 프로그램 | Program | 행사 프로그램 구성, 타임테이블 등 |
| 운영 | Operation | 인력 배치, 동선, 운영 체계 등 |
| 타임라인 | Timeline | 전체 일정 계획, 마일스톤 등 |

#### 섹션별 입력

각 섹션에서 다음과 같은 입력이 가능합니다.

- **텍스트 입력:** 자유 형식의 텍스트 기입
- **동적 리스트:** 항목을 추가/삭제할 수 있는 리스트
- **이미지 업로드:** 이미지 파일 첨부
- **텍스트 주석:** 이미지나 항목에 대한 부연 설명

#### PDF 출력

1. **PDF 출력** 버튼을 클릭합니다.
2. 6개 섹션의 모든 내용이 포함된 PDF 문서가 생성됩니다.

#### PPT 출력

1. **PPT 출력** 버튼을 클릭합니다.
2. pptxgenjs 라이브러리를 통해 PowerPoint 파일이 생성됩니다.
3. 각 섹션이 슬라이드로 구성되어 다운로드됩니다.

> **활용 팁:** PPT 출력 후 디자인 팀에서 추가 편집하여 클라이언트 제출용 자료로 활용할 수 있습니다.

## 7. 내 업무

**내 업무** 메뉴에서는 로그인한 사용자의 개인 업무 현황을 한눈에 확인할 수 있습니다.

### 7.1 담당 프로젝트

- **일반 사용자:** 본인이 **프로듀서(producer)**로 지정된 프로젝트만 표시됩니다.
- **관리자:** 전체 프로젝트가 표시됩니다.

### 7.2 미완료 과업

- 담당 프로젝트 중 **미완료 상태인 체크리스트 항목**이 최대 **10건**까지 표시됩니다.
- 우선순위가 높거나 마감이 임박한 항목이 상단에 배치됩니다.

### 7.3 나에게 위임된 과업

- 다른 팀원이 업무 로그에서 **과업 위임**한 항목 중 나에게 할당된 과업이 표시됩니다.
- 위임자, 과업명, 전달사항을 확인할 수 있습니다.

### 7.4 승인 대기 지출결의/인건비

- 본인이 결의한 **지출결의** 또는 **인건비** 중 아직 승인 처리되지 않은 건이 표시됩니다.
- 항목을 클릭하면 해당 지출결의/인건비 상세로 이동합니다.

### 7.5 내 구매요청

- 본인이 등록한 **구매요청** 목록이 표시됩니다.
- 구매 진행 상태를 확인할 수 있습니다.

## 8. 지출결의

프로젝트별 지출결의를 등록하고 관리합니다.

### 8.1 지출결의 등록

1. **지출** 메뉴로 이동합니다.
2. **+ 새 지출결의** 버튼을 클릭합니다.
3. 아래 항목을 입력합니다.

| 항목 | 필수 | 설명 |
|------|------|------|
| 프로젝트 | 필수 | 관련 프로젝트 선택 |
| 지출내용 | 필수 | 지출 내역 설명 |
| 금액 | 필수 | 지출 금액 (원) |
| 세금형태 | 필수 | 부가세별도 / 부가세포함 / 면세 / 영세율 |
| 은행 | 필수 | 수취 은행 (16종 중 선택) |
| 계좌번호 | 필수 | 수취 계좌번호 |
| 예금주 | 필수 | 계좌 예금주명 |
| 증빙 | 필수 | 세금계산서 / 카드전표 / 현금영수증 / 간이영수증 |
| 결의자 | 필수 | 결의 담당자 |
| 증빙수취 | 선택 | 증빙 수취 여부 |
| 지급상태 | 선택 | 예정 / 지급완료 |

4. **등록** 버튼을 클릭합니다.

### 8.2 세금 자동계산

세금형태를 선택하면 부가세가 자동으로 계산됩니다.

| 세금형태 | 계산 방식 | 예시 (금액 1,000,000원 기준) |
|----------|-----------|------|
| 부가세별도 | 금액 x 1.1 = 합계 | 공급가 1,000,000 + VAT 100,000 = 합계 1,100,000 |
| 부가세포함 | 역산 (합계 / 1.1 = 공급가) | 공급가 909,091 + VAT 90,909 = 합계 1,000,000 |
| 면세 | VAT = 0 | 공급가 1,000,000 + VAT 0 = 합계 1,000,000 |
| 영세율 | VAT = 0 | 공급가 1,000,000 + VAT 0 = 합계 1,000,000 |

### 8.3 지원 은행 목록

다음 16개 은행을 선택할 수 있습니다.

| | | | |
|---|---|---|---|
| 국민은행 | 신한은행 | 우리은행 | 하나은행 |
| 농협은행 | 기업은행 | SC제일은행 | 씨티은행 |
| 케이뱅크 | 카카오뱅크 | 토스뱅크 | 대구은행 |
| 부산은행 | 경남은행 | 광주은행 | 수협은행 |

### 8.4 증빙 유형

| 증빙 유형 | 설명 |
|-----------|------|
| 세금계산서 | 정식 세금계산서 |
| 카드전표 | 법인/개인 카드 매출전표 |
| 현금영수증 | 현금영수증 발행 건 |
| 간이영수증 | 간이영수증 (3만 원 초과 시 비용 인정 제한 주의) |

### 8.5 필터 및 합계

#### 필터

| 필터 옵션 | 설명 |
|-----------|------|
| 전체 | 모든 지출결의 표시 |
| 예정 | 지급 예정인 건만 표시 |
| 지급 | 지급 완료된 건만 표시 |

#### 합계 표시

화면 하단에 다음 합계 정보가 표시됩니다.

- **총 건수:** 조회된 지출결의 건수
- **지급예정:** 아직 지급되지 않은 총 금액
- **지급완료:** 이미 지급된 총 금액
- **지급요금액:** 지급이 필요한 총 금액

## 9. 인건비

프로젝트별 인건비를 등록하고 원천세를 자동 계산합니다.

### 9.1 인건비 등록

1. **인건비** 메뉴로 이동합니다.
2. **+ 새 인건비** 버튼을 클릭합니다.
3. 아래 항목을 입력합니다.

| 항목 | 필수 | 설명 |
|------|------|------|
| 프로젝트 | 필수 | 관련 프로젝트 선택 |
| 내용 | 필수 | 인건비 내역 설명 |
| 금액 | 필수 | 인건비 금액 (원) |
| 세금형태 | 필수 | 사업소득 / 기타소득 |
| 주민번호 | 필수 | 수취인 주민등록번호 |
| 은행 | 필수 | 수취 은행 |
| 계좌번호 | 필수 | 수취 계좌번호 |
| 예금주 | 필수 | 계좌 예금주명 |
| 결의자 | 필수 | 결의 담당자 |
| 보고일 | 선택 | 보고 날짜 |
| 지급상태 | 선택 | 예정 / 지급완료 |

4. **등록** 버튼을 클릭합니다.

### 9.2 원천세 자동계산

세금형태를 선택하면 원천세가 자동으로 계산됩니다.

| 세금형태 | 세율 | 구성 | 예시 (1,000,000원 기준) |
|----------|------|------|------|
| 사업소득 | 3.3% | 소득세 3% + 지방소득세 0.3% | 세액 33,000 / 실지급 967,000 |
| 기타소득 | 22% | 소득세 20% + 지방소득세 2% | 세액 220,000 / 실지급 780,000 |

#### 사업소득 상세 계산

```
총 금액:      1,000,000원
소득세 (3%):     30,000원
지방소득세 (0.3%): 3,000원
원천세 합계:     33,000원
실지급액:       967,000원
```

#### 기타소득 상세 계산

```
총 금액:       1,000,000원
소득세 (20%):    200,000원
지방소득세 (2%):  20,000원
원천세 합계:     220,000원
실지급액:        780,000원
```

### 9.3 합계 표시

- **세액 합계:** 전체 인건비의 원천세 합계
- **실지급 합계:** 전체 인건비에서 원천세를 제외한 실지급 총액

## 10. 구매요청

프로젝트에 필요한 물품/서비스의 구매를 요청하고 처리합니다.

### 10.1 구매요청 등록

1. **구매** 메뉴로 이동합니다.
2. **+ 새 구매요청** 버튼을 클릭합니다.
3. 아래 항목을 입력합니다.

| 항목 | 필수 | 설명 |
|------|------|------|
| 요청일 | 자동 | 요청 등록일 (자동 입력) |
| 마감일 | **필수** | 구매 필요 마감일 |
| 프로젝트 | 선택 | 관련 프로젝트 |
| 구분 | 필수 | 구매 품목 구분 (8종) |
| 품목명 | 필수 | 구매할 품목의 이름 |
| 규격 | 선택 | 품목의 규격/사양 |
| 수량 | 필수 | 구매 수량 |
| 단가 | 필수 | 품목 단가 (원) |
| 구매담당자 | 선택 | 구매 처리 담당자 |
| 구매링크 | 선택 | 구매처 URL |
| 비고 | 선택 | 추가 메모 |

4. **등록** 버튼을 클릭합니다.

### 10.2 구매 품목 구분 (8종)

| 구분 | 설명 |
|------|------|
| 장비 | 음향, 조명, 영상 등 기술 장비 |
| 소모품 | 일회성 소모 물품 |
| 인쇄물 | 현수막, 리플릿, 포스터 등 |
| 소품 | 무대 소품, 장식물 등 |
| 식음료 | 케이터링, 음료 등 |
| 의상 | 유니폼, 의상 등 |
| 서비스 | 외주 서비스, 인력 |
| 기타 | 위 구분에 해당하지 않는 항목 |

### 10.3 합계 자동계산

- **합계금액** = 단가 x 수량
- 합계금액은 단가 또는 수량을 입력/변경하면 자동으로 계산됩니다.

### 10.4 구매 처리

#### 구매완료 처리

- **구매담당자(buyerId)만** 구매완료를 확인할 수 있습니다.
- 구매완료 버튼을 클릭하면 해당 요청이 **완료** 상태로 변경됩니다.

#### 구매 취소

- **요청자 본인** 또는 **관리자**만 구매요청을 취소할 수 있습니다.

### 10.5 긴급 표시 및 D-day

| 표시 | 조건 | 설명 |
|------|------|------|
| 긴급 | 마감 3일 이내 | 마감일이 3일 이내인 경우 빨간색 긴급 표시 |
| D-day | 항상 표시 | 마감일까지 남은 일수 표시 (예: D-5, D-1, D-day) |

> **참고:** 마감일이 지난 구매요청은 **D+N** 형식으로 초과 일수가 표시됩니다.

## 12. 관리자 설정

**관리(AdminPanel)** 메뉴는 **관리자(isAdmin) 권한이 있는 사용자만** 접근할 수 있습니다.

### 12.1 접근 권한 관리

관리자는 시스템 사용자 계정을 관리합니다.

#### 계정 목록 조회

- 전체 등록 계정 목록이 표시됩니다.
- **검색창**으로 사번, 이름 등을 검색할 수 있습니다.

#### 신규 계정 생성

1. **+ 계정 추가** 버튼을 클릭합니다.
2. 사번, 이름, 비밀번호, 권한 등을 입력합니다.
3. **생성** 버튼을 클릭합니다.

#### 계정 승인

1. 상태가 **대기(pending)**인 계정을 찾습니다.
2. **승인** 버튼을 클릭하면 상태가 **승인(approved)**으로 변경됩니다.

#### 계정 차단/해제

1. 차단할 계정의 **차단** 버튼을 클릭합니다.
2. 상태가 **차단(blocked)**으로 변경되어 해당 사용자는 로그인할 수 없습니다.
3. 차단을 해제하려면 **승인** 버튼을 다시 클릭합니다.

#### 계정 편집

1. 편집할 계정의 **편집** 버튼을 클릭합니다.
2. 이름, 비밀번호, 권한 등을 수정합니다.
3. **저장** 버튼을 클릭합니다.

#### 계정 삭제

1. 삭제할 계정의 **삭제** 버튼을 클릭합니다.
2. 확인 대화상자에서 **확인**을 클릭합니다.

> **주의:** `admin` 계정은 **삭제 및 차단이 불가**합니다. 시스템의 최소 1개 관리자 계정을 보장하기 위한 안전장치입니다.

### 12.2 기준 금액 관리

프로젝트 생성 시 자동으로 복사되는 **기준 단가표(priceList)**를 관리합니다.

#### 항목 추가

1. **+ 항목 추가** 버튼을 클릭합니다.
2. 구분, 항목명, 단가, 단위 등을 입력합니다.
3. **저장** 버튼을 클릭합니다.

#### 항목 수정

1. 수정할 항목의 **편집** 버튼을 클릭합니다.
2. 내용을 수정합니다.
3. **저장** 버튼을 클릭합니다.

#### 항목 삭제

1. 삭제할 항목의 **삭제** 버튼을 클릭합니다.
2. 확인 후 삭제됩니다.

#### 구분별 그룹

- 기준 금액 항목은 **구분(기획/테크니컬/제작/기타)**별로 그룹화되어 표시됩니다.
- 각 구분별 소계 및 전체 합계 **통계**가 표시됩니다.

> **참고:** 기준 금액을 변경해도 이미 생성된 프로젝트의 체크리스트에는 영향이 없습니다. 변경된 기준 금액은 **이후 새로 생성하는 프로젝트**에만 적용됩니다.

## 13. 반응형 디자인 및 디바이스별 안내

EventOS는 데스크톱, 태블릿, 모바일 환경 모두에서 사용할 수 있도록 반응형으로 설계되었습니다.

> **팁:** 모바일에서는 빈번한 데이터 입력보다는 **현황 확인** 및 **간단한 코멘트 입력** 위주로 사용하는 것이 편리합니다.

### 13.2 태블릿 (769~1100px)

| 항목 | 동작 |
|------|------|
| 네비게이션 | 상단 탭 표시 |
| 그리드 | **2열** 레이아웃 |
| 카드 | 한 줄에 2개씩 표시 |

### 13.3 데스크톱 (1100px 이상)

| 항목 | 동작 |
|------|------|
| 네비게이션 | 상단 탭 바 표시 |
| 그리드 | **3~4열** 레이아웃 |
| 카드 | 한 줄에 3~4개씩 표시 |
| 전체 기능 | 모든 기능을 최적의 레이아웃으로 사용 |

> **권장:** 견적서 작성, 보고서 편집, 간트차트 관리 등 상세 작업은 **데스크톱 환경**에서 사용하는 것을 권장합니다.

## 부록 B: 트러블슈팅

### B.1 로그인 관련

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 로그인이 되지 않음 | 비밀번호 5회 오류로 잠금 | 10분 후 재시도 |
| "대기 중" 메시지 | 계정이 아직 승인되지 않음 | 관리자에게 계정 승인 요청 |
| "차단된 계정" 메시지 | 관리자가 계정을 차단함 | 관리자에게 차단 해제 요청 |
| 탭을 닫으니 로그아웃됨 | sessionStorage 기반 세션 | 정상 동작. 다시 로그인 필요 |

### B.2 데이터 동기화 관련

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 상태바에 "error" 표시 | 클라우드 동기화 실패 | 네트워크 확인 후 재시도. 로컬 데이터는 보존됨 |
| 상태바에 "offline" 표시 | 네트워크 연결 끊김 | 네트워크 연결 후 자동 동기화됨 |
| 다른 기기에서 데이터가 안 보임 | 동기화 지연 | 3초 이상 대기 후 새로고침 |
| 데이터가 사라짐 | localStorage 삭제됨 | 복원 기능으로 최근 백업에서 복구 |

### B.3 견적서 관련

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 견적 금액이 0원 | 체크리스트에 단가/수량 미입력 | 체크리스트 항목의 단가와 수량 확인 |
| PDF가 생성되지 않음 | 브라우저 팝업 차단 | Chrome 팝업 허용 설정 확인 |
| 공공기관 견적에서 10원 단위 오류 | 정상 동작 (절사 적용) | 10원 단위 절사는 정상적인 계산 방식 |

### B.4 파일 업로드 관련

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|

### B.5 일반 문제

| 문제 | 원인 | 해결 방법 |
|------|------|-----------|
| 화면이 깨져 보임 | 브라우저 호환성 | Chrome 최신 버전 사용 권장 |
| 특정 메뉴가 보이지 않음 | 권한 부족 (관리 메뉴 등) | 관리자 권한 필요. 관리자에게 문의 |
| 모바일에서 표가 잘림 | 작은 화면 크기 | 가로 스크롤로 확인하거나 데스크톱 사용 |
| 변경사항이 저장되지 않음 | 브라우저 저장 공간 부족 | 브라우저 캐시 정리 후 재시도 |

### 2-1. 김지현 (kim) - 행사 기획 팀장 / 총괄 PD

| 항목 | 내용 |
|------|------|
| **이름** | 김지현 |
| **아이디** | kim |
| **비밀번호** | Test1234! |
| **역할/직책** | 행사 기획 팀장 / 총괄 PD |
| **소속 팀** | 기획팀 |
| **권한 수준** | 관리자 (isAdmin=true) |
| **기술 친숙도** | 상 |
| **주요 디바이스** | 데스크톱 (Chrome) + 노트북 (Chrome) |

**주요 업무:**
1. 프로젝트 전체 생성 및 관리 (CRUD)
2. 견적서 작성 및 승인
3. 보고서 작성 및 최종 검토
4. 과업지시서 관리 및 배분
5. 팀원 업무 위임 및 진행 상황 모니터링

**기대 시나리오 (대표 워크플로우):**
- 신규 프로젝트 생성 → 기본 정보 입력 → 체크리스트 템플릿 적용 → 견적서 초안 작성 → 팀원에게 업무 위임 → 간트차트에서 전체 일정 확인 → 보고서 최종 편집 및 출력
- 관리자 패널에서 신규 사용자(한소미) 승인 처리

**잠재적 문제 상황:**
- 동시에 여러 프로젝트를 열고 작업할 때 데이터 혼선
- 견적서 금액이 큰 경우(1억 원 이상) 숫자 포맷 오류
- 관리자 권한과 일반 사용자 권한 전환 시 캐시 문제
- 보고서 출력 시 이미지 레이아웃 깨짐

**담당 테스트 영역:**
- Admin 기능 전반 (사용자 승인, 단가 설정)
- 견적서 생성/수정/출력
- 보고서 작성/편집/출력
- 프로젝트 CRUD (생성, 조회, 수정, 삭제)

3. 체크리스트 기술 디비전 항목 편집
4. 장비 관련 구매요청 작성
5. 기술 스태프 인건비 등록

**기대 시나리오 (대표 워크플로우):**
- 프로젝트 선택 → 체크리스트에서 기술 디비전(음향/조명/영상) 항목 확인 및 수정 → 간트차트에서 셋업-리허설-본행사-철거 일정 세부 조정 → 장비 추가 필요 시 구매요청 작성 → 기술 스태프 인건비 등록 → 업무 로그에 기술 진행 상황 기록

**잠재적 문제 상황:**
- 체크리스트 기술 디비전에 대량의 하위 항목(50개 이상) 추가 시 렌더링 지연
- 간트차트에서 하위 태스크 간 의존성 설정이 올바르게 동작하지 않는 경우
- 장비 단가 데이터가 기준 단가와 동기화되지 않는 경우
- 태블릿에서 간트차트 터치 조작(드래그, 핀치 줌) 오작동
- 기술 용어가 포함된 긴 항목명이 UI에서 잘리는 문제

**담당 테스트 영역:**
- 체크리스트 기술 디비전 항목 관리
- 간트차트 하위 태스크 및 의존성
- 장비 추적 및 구매요청
- 태블릿 환경 간트차트 조작

- 파일 다운로드 권한이 없는 파일에 접근 시도
- 외부 사용자 계정으로 내부 전용 메뉴(관리자 패널 등)가 노출되는 경우
- 협력사 전환(다른 프로젝트 참여) 시 이전 프로젝트 데이터 잔류

**담당 테스트 영역:**
- 프로듀서(producer) 필터 기반 프로젝트 접근 제한
- 외부 사용자 파일 접근 권한
- 제한된 뷰에서의 체크리스트/간트차트 조회
- 보안 및 데이터 격리 검증

3. 전체 시스템 데이터 관리 및 무결성 확인

**담당 테스트 영역:**
- 관리자 패널 전체 기능 (사용자 CRUD, 승인/반려)
- 기준 단가표 관리
- 데이터 무결성 및 백업/복원
- 시스템 보안 (권한 에스컬레이션 방지)

- 모바일과 데스크톱 간 작성 중인 데이터 동기화 지연

**담당 테스트 영역:**
